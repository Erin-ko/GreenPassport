# 綠色循環經濟護照 (Green Cycle Passport) 需求規格書

## 1. 專案概述與目標 (Project Overview & Goals)
- **專案名稱**：綠色循環經濟護照 (Green Cycle Passport)
- **核心目標**：延長物品的使用壽命，極大化資源價值，並減少因遺忘、閒置或小瑕疵造成的物資浪費。
- **解決痛點**：
  - 用房在購入物品後常忽略其保養與使用壽命，容易因忘記而放至過期。
  - 物品出現微小瑕疵時，因缺乏便利的維修資訊或配件，常被直接丟棄。
  - 用戶手邊有低頻率使用的物品（買了只用一下下），缺乏便捷的活化、轉售或捐贈管道。

## 2. 目標用戶與使用場景 (Target Users & Scenarios)
- **主要用戶角色**：一般消費者、周邊鄰里社群
- **典型使用場景**：
  - **場景一：物品登錄與壽命管理**：消費者購買了一張椅子，透過手機登入系統，輸入名稱、材質、購買日期，並設定預估壽命與提醒頻率，建立該物品的「專屬數位護照」。
  - **場景二：物品維護與循環處置**：當椅子即將過期或需要保養時，系統主動發出提醒，並給出低成本延長壽命的維修建議、二手估價、回售分類或捐贈管道建議。
  - **場景三：微型零件社群募資**：消費者的椅子少了一個螺絲，不需要去賣場買一大包，而是直接在系統的互助看板發文詢問周邊鄰居是否有單個閒置螺絲，達到微型資源活化。

## 3. 功能需求與範圍 (Functional Requirements & Scope)
### 核心功能 (MVP)
- [ ] **會員註冊與登入**：支援一般使用者註冊帳號與登入，以安全儲存及讀取個人所屬的物品收藏庫。
- [ ] **物品護照建立與管理（收藏庫）**：
  - 使用者手動登錄物品基本資料：相片、物品名稱、材質、購買日期、預估壽命（到期日期）、維修紀錄（物品歷程）。
- [ ] **到期與保養提醒**：預設壽命到期提醒，或讓使用者自訂定期提醒時間。
- [ ] **健康度評估與維護建議**：
  - 顯示物品健康度。
  - 提供保養建議，並給予如何延長壽命或低成本維修的決策指引。
- [ ] **綠色循環建議**：
  - 出售估價與回售分類。
  - 捐贈管道與地點建議。
- [ ] **微型互助看板**：
  - 用戶可發布簡單貼文募集微型零件（如單個螺絲）或維修協助。
  - 需具備隱私保護的位置範圍（如僅顯示距離，不透露精確地址）。

### 進階/未來功能 (Out of Scope / Future Phases)
- **永續與健康度進度條**：隨時間經過，動態更新物品健康度進度條，並於生命週期結束時評估其永續分數。
- **年度 ESG 成就報告**：統計使用者年度維修、出售、捐贈、共享的次數，並估算延長之壽命與減碳量，支持成果分享。
- **AI 影像材質辨識**：自動辨識照片中的物品與材質以加速登錄。

## 4. 技術與非功能需求 (Technical & Non-functional Requirements)
- **技術棧**：
  - 前端：純 HTML/CSS/JS (Vanilla 多頁面網頁)，以便使用者能直接在資料夾編輯網頁。
  - 後端：FastAPI (Python)
  - 本地資料庫：SQLite
  - 正式資料庫：Render PostgreSQL
  - 部署平台：Render.com (採用免費方案)
- **系統平台**：響應式網頁設計 (RWD)，需同時優化電腦版與手機版介面。
- **非功能需求**：
  - **安全性與隱私保護**：
    - 使用者個人上傳資料需受到存取權限控管。
    - 互助看板僅呈現模糊化位置或距離範圍（如「距離 500m」），嚴格保護精確住址。
  - **效能**：頁面加載迅速，適合手機行動網路存取。

## 5. 專案時程與限制條件 (Timeline, Budget & Constraints)
- **預計時程**：3 天內完成概念 Demo/原型系統。
- **限制條件**：
  - 預算限制：所有雲端部署與第三方服務必須使用**免費方案**。
  - 技術時程緊湊，功能需聚焦於 MVP，暫不實作第二階段之 ESG 統計與動態進度條。
